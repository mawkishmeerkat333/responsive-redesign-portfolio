Responsive Redesign

by Mawkish Meerkat

Introduction

For this project I redesigned a page from the website of a local restaurant in my hometown. To do so, I identified usability and accessibility concerns with the original design, drew up wireframes, created a style guide, prototyped my redesign, and designed the new webpage with HTML, CSS, and a sprinkle of JavaScript. This was a fun exercise in considering all aspects that go into frontend website design, so enjoy!

Part 1: Identify Problems

Part 1a: Picking a Website

I knew from the outset that I wanted to redesign a website related to something in my hometown, so I went with the landing page of the website for a local restaurant named Amalfi's. The webpage was just right: it had enough issues upon which I felt comfortable improving while not so many that I didn't know where to start. You can check out for the original webpage here.

Part 1b: Usability Problems

After looking over the webpage—clicking buttons, resizing the window, following links—I found numerous issues:

  1. The title is actually an image that can bleed off-screen and create awkward white margins.
  2. The website is inconsistent in styling its buttons: there are four different designs used. This can create confusion.
  3. Similarly, there are too many different fonts: at least five that are apparent.
  4. There are two buttons for ordering online: one amongst the navbar buttons and another next to a button for DoorDash Delivery. I think this is in conflict with the user's conceptual model. The navbar usually contains internal navigation links, so linking to an external ordering service there would be clunky, even if there were not already another button for ordering online.
  5. Moreover, the button to call is not grouped alongside the two order buttons in desktop or mobile. It would be more intuitive to group all ordering methods together.
  6. Amalfi's landing page does not make use of its footer. There is a separate page for contact and reservations, which works, but many sites also list hours of operation, social media, and such in the footer.
  7. In desktop view, two of the buttons clip off-screen.
  8. When viewed on a phone, the ordering is out of sorts. The online order and DoorDash buttons appear first, then the welcome text "Benvenuto!", then an awkwardly large space, then the "Call Now!" button, then a picture, text, and a second picture. This ordering breaks up the logical flow and sensible groupings of the elements.
Part 1c: Accessability Problems

Looking at the accessibility concerns flagged by WAVE, it confirmed some of my thoughts as well as pointed out issues I had missed:

  1. There is an empty heading on the page: this is the culprit behind the weird spacing in mobile view.
  2. WAVE warned of low contrast with the welcome text, something I noticed when I first visited the site.
  3. Furthermore, WAVE also warned oflow contrast with the navbar buttons. I think the contrast issue with the buttons is less egregious, but the buttons are a tad unsightly, so the buttons need work either way.
  4. WAVE noted bad alt text and ironically some of the alt text it labeled as good was just as bad. *Untitled* is not good alt text WAVE!
  5. WAVE flagged one of the buttons for "suspicious link text" because it contained the phrase "click here". In its explanation of the alert, WAVE considers text like "click here" as extraneous. This is not something I initially considered, but if the element is clearly a link, that phrasing is indeed redundant.
  6. Finally, WAVE alerted me to very small text. I couldn't find the text until I turned off styling, because the div containing the text has display: none. It seems to be intended for screen readers only, so it is otherwise hidden to the naked eye. The note seems unneeded and so could be done away with.

Part 2: Visual Redesign

Part 2a: Lo-fi Wireframing

After taking note of the aforestated issues in the exploratory of this project, I moved on to drafting the layout and styling of my redesign. First up was to create low-fidelity wireframes for the three screen sizes under consideration—mobile, tablet, and desktop.

With a general idea of what elements I wanted to include, I went ahead creating a cohesive design that addressed the issues such as too many fonts, poor contrast, and inconsistent button styling.

Part 2b: Visual Design Style Guide

The style guide depicted is the final iteration, which includes a few additions and updates. First, the text inside the buttons was initially the off-white color. However, WAVE flagged that for low contrast. I changed it to the off-black, and not only is the contrast now better, but I also think the final button design looks nicer. After deciding to include a footer while building the website, I paused to consider how to style hyperlinks and the social media logos in order to maintain cohesion.

Part 2c: Hi-fi Prototyping

I finally moved on to the high-fidelity prototypes, which I have retroactively updated to use the the new text color for the buttons as well as adjust a comment or two to reflect the reality of how I coded the webpage.

Takeaways

Even though my redesign will not become the new site for Amalfi's, the whole process has felt a bit like giving back to my community. As I get better at website design throughout this class and beyond, perhaps I can offer to create/redesign websites for those in my community pro bono.
